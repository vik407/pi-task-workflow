# Project Patterns

Record reusable project patterns discovered during analysis, planning, or implementation.

Each entry should include:

- Context.
- Pattern.
- Files or modules that demonstrate it.
- When to use it.
- When not to use it.

## Rules

- Do not rediscover stable facts on every task.
- Reuse known stack, architecture, commands, conventions, environments, and validation paths.
- Update patterns only when new evidence changes them.
- Keep entries short and practical.
- Prefer facts learned from real project files or completed tasks.
